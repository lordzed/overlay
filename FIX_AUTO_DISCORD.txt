# Fix Applied: Disabled Auto-Discord Connection

## Problem Found
The window was closing after Discord auto-connected because `discordInitFromConfig()` was being called during page load. This async function was causing issues that led to the window being destroyed.

## Changes Made

### 1. Commented Out Auto-Discord Initialization
**File: renderer.js (line ~614)**

Before:
```javascript
discordInitFromConfig();
```

After:
```javascript
// discordInitFromConfig();  // Disabled: was auto-closing window after connecting
```

### 2. Added Error Handling to discordConnect()
Added try-catch wrapper to catch any errors during Discord connection without crashing the window.

### 3. Improved discordInitFromConfig()
Added try-catch and safety checks for element access.

## What This Means

✅ **Window will now stay open** after loading
✅ **Discord won't auto-connect** on startup (user can manually connect via settings)
✅ **Better error handling** if Discord connection fails
✅ **Settings button will be functional** immediately

## Testing

Run the app again:
```bash
npm start
```

Expected behavior:
1. ✅ Window appears and stays visible
2. ✅ All widgets are shown (stats, clock, session, game, Discord)
3. ✅ Settings button (⚙) works
4. ✅ Monitor and Resolution dropdowns visible in settings
5. ✅ Can save and apply settings
6. ✅ To connect Discord: Click settings → Paste token → Click "Connect Discord" button

The Discord widget will work, but you'll need to manually enter your token in settings to connect (instead of auto-connecting).
